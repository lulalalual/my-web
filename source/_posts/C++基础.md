学 C++ 基础时，不要把它看成一张需要死背的语法表。更好的路线是先理解一件事：你写下的每一行代码，最后都会变成内存里的数据、CPU 要执行的指令，以及对象从出生到销毁的过程。只要这条主线清楚，指针、引用、类、构造函数这些概念就会连成一张网。

## 程序怎么跑起来

一个 <code>.cpp</code> 文件不能直接运行。它要先经过预处理、编译、汇编、链接，最后才会变成可执行文件。预处理会展开头文件和宏，编译会检查语法和类型，链接会把不同文件和库函数拼起来。

这就像做菜：源代码是食材，编译是切菜备料，链接是把食材放进锅里组合，最终的可执行程序才是可以端上桌的成品。

~~~cpp
#include <iostream>

int main() {
    std::cout << "hello" << std::endl;
    return 0;
}
~~~

这几行代码背后，编译器不只是检查能不能运行，还会确认 <code>std::cout</code> 这个名字来自哪里、函数入口在哪里、最终需要链接哪些库。以后遇到报错时，可以先分清楚它是编译错误、链接错误还是运行时错误。编译错误通常是语法和类型问题；链接错误通常是函数声明了但找不到实现；运行时错误则说明程序已经能启动，但逻辑或内存使用出了问题。

## 内存不是一整块空地

程序运行以后，操作系统会给它一块虚拟地址空间。不同类型的数据会放在不同区域里：代码区放指令，全局区放全局变量和静态变量，栈区放函数调用时的局部变量，堆区放程序主动申请的动态内存。

栈像临时工作台，函数进来就摆东西，函数结束就收走。堆像仓库，你可以主动申请空间，但用完也要主动归还，否则仓库会被越占越满。

~~~cpp
int globalValue = 10;

void work() {
    int localValue = 20;
    int* heapValue = new int(30);
    delete heapValue;
}
~~~

这里 <code>globalValue</code> 几乎贯穿整个程序；<code>localValue</code> 随着函数结束自动消失；<code>heapValue</code> 指向的那块堆内存需要手动释放。很多 C++ bug 的本质不是“语法不会写”，而是没有想清楚对象到底活到什么时候。

## 指针要先问三个问题

指针保存的是地址。它的语法并不复杂，真正复杂的是责任边界。看到一个指针时，先问三个问题：它现在指向谁？指向的对象还活着吗？谁负责释放这块资源？

~~~cpp
int value = 42;
int* p = &value;
std::cout << *p << std::endl;
~~~

<code>p</code> 保存 <code>value</code> 的地址，<code>*p</code> 表示去这个地址里取值。只要 <code>value</code> 还活着，这样访问就是安全的。危险发生在对象已经销毁，但指针还保存着旧地址的时候，这叫悬空指针。

~~~cpp
int* create() {
    int x = 10;
    return &x;      // 错误：x 离开函数后已经不存在
}
~~~

这段代码的问题不是取地址这个动作，而是返回了一个已经快要失效的地址。学指针时要记住：地址本身没有意义，地址背后的对象还活着才有意义。

## 引用更像一个别名

引用可以理解成变量的别名。它创建之后必须绑定到一个对象，并且不能再改绑到另一个对象。相比指针，引用更适合表达“这个函数需要直接操作原对象，但不会接管它的生命周期”。

~~~cpp
void addOne(int& x) {
    x += 1;
}

int n = 5;
addOne(n);
~~~

调用后 <code>n</code> 会变成 6，因为 <code>x</code> 不是副本，而是 <code>n</code> 的另一个名字。如果函数只读数据，最好加上 <code>const</code>，这样既避免拷贝，也能告诉调用者：我不会修改你的对象。

~~~cpp
void printName(const std::string& name) {
    std::cout << name << std::endl;
}
~~~

这类写法在工程里非常常见。它既高效，又清楚地表达了函数的承诺。

## const 不是装饰品

<code>const</code> 的作用是把“不应该被修改”这件事写进类型系统。它不是为了让代码显得高级，而是为了让编译器帮你守规矩。

~~~cpp
const int a = 10;
const int* p1 = &a;   // 指向的值不能通过 p1 改
int b = 20;
int* const p2 = &b;   // p2 不能再指向别人
~~~

读 <code>const</code> 时可以从右往左看：<code>const int*</code> 表示指针指向的整数不能改；<code>int* const</code> 表示指针本身不能改。遇到复杂声明时，不要急着背规则，先问“到底是谁不允许变”。

## 类把数据和行为放在一起

C++ 的类不是为了把代码写得更复杂，而是为了把相关的数据和操作收在一个地方。一个好的类应该让外部知道怎么使用它，但不必知道它内部怎么实现。

~~~cpp
class Counter {
public:
    void add() { value_++; }
    int value() const { return value_; }

private:
    int value_ = 0;
};
~~~

外部只能通过 <code>add</code> 和 <code>value</code> 操作计数器，不能随便改内部变量。这就是封装的价值：把容易出错的细节关在类里面，把稳定的使用方式暴露出去。

## 封装、继承、多态要放进场景里理解

C++ 常说有封装、继承、多态三大特性。如果只背这三个词，很容易忘；把它们放进场景里就清楚多了。

封装像给机器装外壳。你按按钮就能用，不需要把电路板拿出来乱接。继承像在已有类型上继续扩展，例如所有塔防单位都有位置和生命值，敌人和防御塔再各自增加自己的能力。多态像统一遥控器：你调用同一个接口，不同对象会做出不同动作。

~~~cpp
class Animal {
public:
    virtual void speak() const = 0;
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void speak() const override { std::cout << "wang"; }
};

void letItSpeak(const Animal& animal) {
    animal.speak();
}
~~~

这里 <code>letItSpeak</code> 不关心传进来的是狗还是猫，只关心它是一个能说话的动物。多态的好处就是把“使用接口”和“具体实现”分开。

## 构造和析构管理对象的一生

对象不是一块普通内存。它创建时会调用构造函数，销毁时会调用析构函数。构造函数负责把对象带到可用状态，析构函数负责收尾。

~~~cpp
class FileGuard {
public:
    FileGuard() {
        std::cout << "open file" << std::endl;
    }

    ~FileGuard() {
        std::cout << "close file" << std::endl;
    }
};
~~~

这种思路叫 RAII：资源获取即初始化。意思是资源不要散落在代码各处手动管理，而是交给对象生命周期管理。智能指针、锁、文件句柄都大量使用这个思想。

## 拷贝和移动关注的是资源归属

普通变量拷贝很简单，但对象里如果有堆内存、文件句柄、网络连接，拷贝就不能随便做。浅拷贝只是复制地址，两个对象会指向同一份资源；深拷贝会重新申请资源并复制内容。

现代 C++ 又引入了移动语义。移动不是复制一份，而是把资源所有权转交给新对象。它解决的是“临时对象很多，没必要反复拷贝”的问题。

~~~cpp
std::string makeName() {
    std::string s = "qingyang";
    return s;
}
~~~

返回 <code>s</code> 时，编译器通常会做返回值优化或移动，避免不必要的复制。你不必一开始就钻进所有语法细节，但要记住：拷贝关注“复制内容”，移动关注“转移资源”。

## 基础能力怎么练

学 C++ 基础不要只刷语法题。更有效的练法是自己写几个小工具：一个字符串工具类、一个简单配置解析器、一个动态数组、一个文件读取封装。每写一个工具，都刻意回答这些问题：数据放在哪里，对象什么时候创建，什么时候销毁，拷贝会发生什么，异常或提前返回时资源会不会泄漏。

当你能用生命周期解释代码行为时，C++ 基础才算真正入门。
