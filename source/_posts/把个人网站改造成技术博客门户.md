这个站点的核心目标很明确：展示项目，也持续记录笔记。相比复杂的在线后台，博客型信息架构更稳定，也更适合长期维护。

## 为什么切换到 Hexo

Hexo 的优势是内容天然由 Markdown 驱动，文章、分类、标签、归档都可以在构建时直接生成。AnZhiYu 主题已经提供了首页推荐区、双栏文章流、右侧信息栏、标签云、归档、暗色模式和动效能力。

这意味着网站不再依赖数据库服务，维护流程变成：

```bash
npm run dev
npm run build
git add .
git commit -m "更新文章"
git push
```

Vercel 监听 GitHub 仓库，推送后会自动重新构建并发布。

## 首页结构

首页采用技术博客门户结构：

- 顶部导航负责文章、分类、标签、项目和关于页。
- 首屏 `home_top` 展示站点标题、分类入口和项目 banner。
- 主内容区展示最近文章。
- 右侧栏展示作者说明、公告、最近文章、分类、标签云和归档。

这种结构比单页作品集更适合持续添加内容，也更接近成熟技术博客的浏览习惯。

## 后续维护

所有新笔记都写在 `source/_posts/`。如果要增加一个项目专栏，可以继续使用 Markdown 文章，也可以在 `source/projects/index.md` 中增加项目卡片入口。
