这个网站现在的定位很简单：一边展示项目，一边把学习和代码复盘写成长期笔记。

## 为什么改

单页作品集适合一次性展示成果，但不适合长期更新。技术博客更像一个可持续生长的仓库：今天写一篇算法，明天补一个项目复盘，内容会慢慢连起来。

## 怎么维护

文章都写在这里：

```text
source/_posts/
```

新增或修改后执行：

```bash
npm run build
git add .
git commit -m "更新文章"
git push
```

推送到 GitHub 后，部署平台会重新生成网页。

## 页面结构

- 首页展示最近文章和推荐入口。
- 专栏按 C++、算法、AI、项目来分。
- 项目合集负责放项目复盘。
- 关于页说明这个网站是谁在维护、写什么内容。

这个结构不复杂，但适合长期写下去。
