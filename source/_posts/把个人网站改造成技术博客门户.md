这个网站一开始更像作品展示页，视觉冲击有了，但不适合长期写内容。后来我把它改成技术博客门户：项目还在，但核心变成文章、专栏和复盘。

## 先说人话

个人网站不能只靠首页撑场面。首页再好看，如果后面没有持续更新的内容，很快就会变成一个静态名片。

我更想要的是一个能长期生长的技术档案：学习 C++ 时能写笔记，做项目时能写复盘，改网站时也能把过程记录下来。

~~~text
作品展示 -> 技术博客 -> 项目档案 -> 长期知识库
~~~

这次改版就是围绕这个目标做的。

## 为什么从作品集改成博客

单页作品集适合快速展示“我做过什么”，但它有几个问题：

- 内容多了以后，页面会越来越长。
- 新增文章不方便，容易影响首页布局。
- 项目只能展示结果，很难讲清楚过程。
- 搜索、分类、标签和归档能力弱。

技术博客刚好补这些短板。每篇文章独立维护，专栏负责分类，标签负责关联，项目复盘负责讲过程。

## 为什么选 Hexo

Hexo 的优点是简单。文章写 Markdown，构建后生成静态页面，部署平台只需要托管生成结果。

它适合当前需求：

- 不需要复杂数据库。
- 不需要在线多人协作后台。
- Git 可以记录每次文章修改。
- 静态页面访问稳定，部署成本低。

如果后面要做评论、搜索增强或会员系统，可以再接服务端。现在先把内容和体验做好。

## 为什么选 AnZhiYu

AnZhiYu 本身就是面向博客的主题，已经有首页、文章页、分类、标签、右侧栏、移动端导航这些基础结构。

我主要做的是风格化改造：深色主题、星空背景、卡片封面、悬停过渡、个人头像和项目入口。

这样比从零写主题更稳，因为博客的基本能力不用重造，只需要把视觉和内容组织改成自己的风格。

## 内容结构怎么定

文章统一放在：

~~~text
source/_posts/
~~~

封面统一放在：

~~~text
source/img/covers/notes/
~~~

每篇文章用 frontmatter 管理元信息：

~~~yaml
title: C++基础
permalink: posts/cpp-basic/
categories:
  - C++语法笔记
tags:
  - C++
  - 指针
cover: /img/covers/notes/cover-cpp-basic.png
description: 用人话讲 C++ 基础。
~~~

这样页面可以自动读取标题、分类、标签和封面，不需要手动维护每一张卡片。

## 分类怎么设计

现在内容主要分成几条线：

- C++：基础、STL、高级、多线程、网络编程。
- 算法：数据结构、算法总览、位图和布隆过滤器。
- 八股：操作系统、计网、MySQL、SQL。
- 项目：面试大师、塔防双人、通信平台、Raft KV。
- 技术笔记：学习导航、改站记录、设计模式等。

分类不能太细。太细会让导航复杂，太粗又找不到内容。现在这个粒度比较适合个人博客。

## 视觉风格怎么取舍

我想要的网站气质是“深色、星空、技术档案、卡片感”。所以背景用了动态星空和流光，文章卡片用了大字封面，右侧栏保留个人头像和公告。

但视觉不能影响阅读。比如悬停动画要丝滑，但不能突然大面积变色；背景可以动，但不能让文字看不清；移动端可以简化布局，但不能丢导航。

好看的前提是可读。

## 动效怎么控制

动效主要分三类：

- 背景动效：星星漂移、光点移动、流光变化。
- 交互动效：按钮悬停、卡片上浮、导航切换。
- 页面动效：首屏出现、卡片渐入、滚动反馈。

动效不是越多越好，而是要让用户感觉页面有生命力。速度太快会晕，速度太慢会拖。现在的策略是：背景慢慢动，悬停有过渡，卡片轻微浮动。

## 移动端怎么处理

移动端最重要的是不溢出、不遮挡、能点到。之前移动端出现过导航和卡片布局不舒服的问题，所以后面重点做了：

- 顶部导航收缩。
- 首页入口卡片改成更适合窄屏的排列。
- 文章卡片避免文字过长挤出。
- 右侧栏在移动端减少干扰。

PC 端可以展示更多装饰，移动端必须优先保证阅读。

## 部署怎么维护

常规更新流程是：

~~~bash
npm run build
git add .
git commit -m "更新文章"
git push
~~~

推送后，部署平台会重新构建。静态站的优势是线上没有复杂运行时，故障点少。

如果部署到国内平台，要重点检查：Node 版本、构建命令、输出目录、postinstall 脚本权限、图片路径和大小写路径。

## 这次改版的收获

这次改版让我更明确：个人网站真正重要的是可持续维护。首页风格可以慢慢调，但内容组织必须稳定。文章要能长期更新，项目要能逐步复盘，封面和导航要能随着内容增长继续工作。

## 总结

这个站点现在不是单纯作品集，而是青阳寻雪的技术档案。它会继续记录 C++、算法、八股、项目和网站改造过程。页面负责提供氛围，文章负责留下真正有价值的内容。
