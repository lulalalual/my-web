位图和布隆过滤器都在解决同一类问题：当数据量非常大时，怎样用更少内存判断一个元素是否出现过。它们牺牲的是部分表达能力，换来的是极高的空间效率。

## 位图把一个数字映射到一个 bit

如果要记录 0 到 10000000 里哪些数字出现过，用普通布尔数组会浪费空间。位图的思路是用一个 bit 表示一个数字是否存在。bit 只有 0 和 1，非常省内存。

假设数字 5 出现了，就把第 5 个 bit 置为 1；要判断 5 是否出现过，就检查第 5 个 bit。

~~~cpp
std::vector<unsigned char> bitmap((maxValue + 8) / 8);

void setBit(int x) {
    bitmap[x / 8] |= (1 << (x % 8));
}

bool exists(int x) {
    return bitmap[x / 8] & (1 << (x % 8));
}
~~~

这里 <code>x / 8</code> 找到它在哪个字节，<code>x % 8</code> 找到它在这个字节里的第几位。

## 位图适合整数范围明确的场景

位图很省空间，但它要求元素能映射到确定范围。比如用户 ID、状态码、整数集合都适合。字符串、对象、范围特别大且稀疏的数据，就不一定适合直接位图。

位图常用于去重、快速存在性判断、海量整数排序、权限标记等场景。它的优点是快和省；缺点是表达能力有限，不能直接存复杂对象，也不方便记录重复次数。

## 布隆过滤器允许少量误判

布隆过滤器比位图更灵活。它通过多个哈希函数把一个元素映射到多个 bit 位。加入元素时，把这些位置都置为 1；查询元素时，如果这些位置都为 1，就认为它可能存在。

注意这里是“可能存在”。因为不同元素可能把同一些 bit 置为 1，所以布隆过滤器会有误判。它不会把一个真实存在的元素判断成不存在，但可能把一个不存在的元素判断成存在。

这就像门口保安看特征判断人有没有来过。如果几个特征都对上了，可能是本人，也可能只是撞特征。但如果某个特征完全没出现，那就一定没来过。

## 布隆过滤器适合挡住明显不存在的数据

布隆过滤器常用于缓存穿透防护。比如数据库里有一亿个合法 ID，用户请求某个 ID 时，先问布隆过滤器。如果它说一定不存在，就不用查缓存和数据库；如果它说可能存在，再继续查。

这样能挡住大量恶意请求或无效请求。即使有少量误判，也只是多查一次后端，不会返回错误数据。

## 哈希函数数量不是越多越好

哈希函数越多，误判率可能下降，但每次插入和查询成本也会上升，bit 位也会更快被置满。如果 bit 位大部分都是 1，过滤器就失去意义了。

工程里通常根据预计元素数量和可接受误判率来计算位数组大小和哈希函数数量。不要凭感觉随便设。

## 删除元素为什么麻烦

普通布隆过滤器不支持安全删除。因为一个 bit 可能由多个元素共同置为 1，如果删除某个元素时直接把对应 bit 清零，可能会误伤其他元素。

如果确实需要删除，可以使用计数布隆过滤器。它不是用 bit，而是用小计数器记录某个位置被多少元素命中。删除时计数减一。但这样会增加内存成本。

## 位图和布隆过滤器怎么选

如果元素是范围明确的整数，并且你需要准确判断，用位图。如果元素类型复杂、数据量巨大，并且能接受“可能存在”的误判，用布隆过滤器。

它们的共同思想是：不保存完整数据，只保存足够回答问题的信息。这个思想在高性能系统里很常见。
