这个项目可以理解成一个小型通信底座：客户端连进来，服务端收消息、转发消息、找服务、存数据。

## 项目做什么

它把几件常见后端能力串在一起：

```text
网络连接 -> RPC 调用 -> 服务发现 -> 消息发布 -> 数据保存
```

用户看到的是聊天或调用结果，背后其实是一条完整链路。

## 核心模块

- Muduo 负责网络连接和事件循环。
- RPC 负责把“调用函数”变成“跨机器请求”。
- ZooKeeper 负责发现服务在哪里。
- Redis Pub/Sub 负责跨节点消息通知。
- MySQL 负责保存用户、好友、群组和离线消息。

## 小代码

```cpp
server.onMessage([&](Connection conn, Message msg) {
    auto service = router.find(msg.type);
    service->handle(conn, msg);
});
```

真实项目会更复杂，但核心思路就是：收到消息，找到处理者，再把结果发回去。

## 收获

这个项目最有价值的地方不是某个库，而是把一条后端链路跑通。后面复盘时，可以继续拆连接管理、消息路由、服务注册和数据一致性。
