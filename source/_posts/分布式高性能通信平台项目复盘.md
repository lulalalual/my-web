`分布式高性能通信平台` 是一个 C++ 分布式通信系统。它的特点是“框架 + 业务”结合：底层实现 RPC 通信框架，上层用即时通讯业务验证框架的可用性。

## 项目定位

项目由两条线融合而来：

- RPC 框架线：提供远程调用、服务发布、服务发现、序列化和异步日志。
- IM 业务线：提供登录、注册、好友、群组、单聊、群聊、离线消息和跨服务器推送。

这种设计的好处是简历表达更完整：既能讲底层网络编程，也能讲业务系统设计。

## 核心架构

系统可以拆成三层：

- 客户端层：IM 客户端、RPC 客户端、管理后台。
- 服务层：ChatServer 集群与 RPC Provider 集群。
- 中间件层：ZooKeeper、Redis、MySQL。

RPC 层负责服务调用，IM 层负责真实业务，中间件层负责服务发现、跨服务器消息和数据持久化。

## RPC 框架

项目使用 Muduo 作为网络框架，采用 Reactor 模型处理高并发连接。RPC 协议使用 Protobuf 序列化，并自定义了消息格式：

```text
[4 字节 header_size][protobuf_header][protobuf_args]
```

这个协议解决了 TCP 粘包/半包问题。接收端先读固定长度的 header size，再读完整 header，最后根据 args size 读取参数体。

## 服务注册与发现

服务提供者启动时向 ZooKeeper 创建临时节点：

```text
/dhc_comm_service/UserService/Login -> 192.168.1.100:9000
```

临时节点和心跳机制让服务宕机后可以自动下线。服务消费者查询节点后直连 provider 发起 RPC。

## IM 与跨服务器通信

IM 业务使用 Redis Pub/Sub 解决多台 ChatServer 之间的消息路由问题：

- 用户登录后订阅自己的频道。
- 发送消息时先查本机连接。
- 如果目标用户不在当前服务器，就发布到 Redis 对应频道。
- 目标用户所在服务器收到订阅消息后推送给本地连接。

这个设计让 ChatServer 可以水平扩展。

## 工程亮点

- Muduo Reactor：主从 EventLoop 分工，减少锁竞争。
- ZooKeeper 临时节点：服务健康检测和发现。
- Redis Pub/Sub：跨服务器实时消息推送。
- MySQL：用户、好友、群组和离线消息持久化。
- LockQueue：生产者消费者模式实现异步日志。
- Config 单例：统一加载配置文件。

## 复盘结论

这个项目的价值在于技术链条完整：网络 IO、协议设计、服务治理、消息中间件、数据库建模和业务分发都有落点。后续可以继续补充连接池、消息 ACK、历史消息漫游、TLS 加密和压测报告。
