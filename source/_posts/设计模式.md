## 通俗导读

设计模式不是固定答案，而是一批常见代码问题的经验解法。它们大致分三类：怎么创建对象、怎么组织对象、怎么让对象协作。

```text
创建型：对象怎么造
结构型：对象怎么拼
行为型：对象怎么沟通
```

## 先看例子

```cpp
class Logger {
public:
    static Logger& instance() {
        static Logger logger;
        return logger;
    }

    void log(const std::string& msg) {
        std::cout << msg << '\n';
    }

private:
    Logger() = default;
};
```

这是单例模式的常见写法：全局只需要一个日志对象，就把创建入口收束到 `instance()`。它属于创建型模式。

## 三类模式怎么理解

- 创建型：关心对象怎么创建，比如单例、工厂、建造者。
- 结构型：关心对象怎么组合，比如适配器、装饰器、代理。
- 行为型：关心对象怎么协作，比如观察者、策略、责任链。

## 阅读建议

学设计模式不要先背名字。先看代码里有没有重复创建、重复 if-else、对象耦合太死这些问题，再选择合适的模式。
