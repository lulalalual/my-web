动态规划适合解决“最优子结构 + 重叠子问题”问题，但真正卡人的地方从来不是代码，而是状态定义。

## 四步法

- 明确状态：`dp[i]` 或 `dp[i][j]` 到底表示什么。
- 明确选择：当前状态由哪些更小状态转移而来。
- 写出转移：把选择关系翻译成方程。
- 确定 base case：最小规模问题的答案是什么。

## 经典一维状态

```cpp
vector<int> dp(n, 1);
for (int i = 0; i < n; ++i) {
    for (int j = 0; j < i; ++j) {
        if (nums[j] < nums[i]) {
            dp[i] = max(dp[i], dp[j] + 1);
        }
    }
}
```

这段代码的意义不是 LIS 本身，而是告诉我们：`dp[i]` 通常表示“以 i 结尾”的最优值。

## 设计状态时常问自己

- 状态里是否包含了决策所需的全部信息。
- 是否把无关信息也塞进了状态，导致维度爆炸。
- 这个状态能否由更小规模状态唯一转移出来。

## 常见类型

- 线性 DP：爬楼梯、打家劫舍、最长递增子序列。
- 背包 DP：容量约束下的选与不选。
- 区间 DP：答案依赖子区间。
- 树形 DP：状态在树上自底向上传递。
- 状态压缩 DP：集合状态用二进制表示。

## 常见错误

- `dp` 定义模糊，导致转移方程写不出来。
- base case 只写了值，没处理越界。
- 以为所有 DP 都必须写二维数组，忽略空间压缩。
- 把贪心问题硬写成 DP，复杂度不必要地升高。
