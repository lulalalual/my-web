图论题最怕一上来就想算法名字。正确顺序应该是：先建模，再遍历，最后根据边权和目标选择 BFS、DFS、拓扑排序或最短路。

## 图的基本问题

- 点表示什么实体。
- 边表示什么关系。
- 图是否有向。
- 边是否带权。

## BFS 适用场景

- 无权图最短路。
- 分层搜索。
- 扩散类问题。

```cpp
queue<int> q;
vector<int> dist(n, -1);
q.push(start);
dist[start] = 0;

while (!q.empty()) {
    int cur = q.front();
    q.pop();
    for (int nxt : graph[cur]) {
        if (dist[nxt] != -1) continue;
        dist[nxt] = dist[cur] + 1;
        q.push(nxt);
    }
}
```

## DFS 适用场景

- 连通块统计。
- 是否存在路径。
- 搜索所有可达节点。
- 递归地处理树与图结构。

## 拓扑排序

拓扑排序只适用于有向无环图。它适合：

- 课程安排。
- 任务依赖。
- 构建顺序。

核心思路是维护入度为 0 的点，不断弹出并删边。

## 做题顺序

- 先画出抽象图。
- 判断是否有环、是否带权、目标是不是最短路。
- 再决定 BFS、DFS、拓扑排序还是 Dijkstra。
