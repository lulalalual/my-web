图论题不要一上来就想算法名。先问四个问题：点是什么，边是什么，有没有方向，有没有权重。

## 怎么建图

```cpp
vector<vector<int>> graph(n);
graph[a].push_back(b);
```

这表示从 `a` 可以走到 `b`。如果是无向图，还要再加一条 `b` 到 `a`。

## BFS

BFS 像一圈一圈往外扩，适合无权图最短路。

```cpp
queue<int> q;
q.push(start);
while (!q.empty()) {
    int cur = q.front();
    q.pop();
}
```

## DFS

DFS 像一条路走到底，适合连通块、路径搜索和递归处理。

## 拓扑排序

拓扑排序适合任务依赖。比如课程 A 学完才能学课程 B，就可以把它看成一条有向边。

## 做题顺序

先把图画清楚，再决定用 BFS、DFS、拓扑排序还是最短路。建模对了，代码才有方向。
