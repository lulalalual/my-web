计算机面试大师是一个 AI 面试训练平台。它不是简单的聊天框，而是把面试前准备、面试中回答、代码题练习、面试后复盘放到一条完整链路里。

## 项目先说人话

真实面试最难的不是“知道一点”，而是能不能在压力下把问题讲清楚。这个项目就是想模拟这种压力：AI 扮演面试官，用户回答，系统继续追问，最后生成报告。

~~~text
选择方向 -> AI 提问 -> 用户回答 -> 继续追问 -> 代码练习 -> 生成复盘
~~~

它的核心价值是让练习变成闭环。没有记录的练习容易忘，有复盘的练习才能知道自己哪里反复出错。

## 技术栈

项目是前后端一体的现代前端应用，主要技术包括：

- React 18 和 TypeScript：负责页面和类型约束。
- Vite：负责开发和构建，启动速度快。
- Tailwind CSS：负责整体 UI 风格。
- Framer Motion：负责页面过渡、卡片动效和能力雷达动效。
- Zustand：管理用户信息和面试会话状态。
- React Router：负责页面路由和代码分割。
- Monaco Editor：提供代码编辑器体验。
- pdfjs-dist：在浏览器端解析简历 PDF。
- Express/Vercel Functions：服务端代理 AI 请求，避免前端暴露 API Key。
- IndexedDB：保存面试记录、错题和本地数据。

这些技术不是为了堆名词，而是各自解决一个问题：页面要快，交互要顺，数据要能保存，AI Key 不能暴露。

## 核心模块

### AI 面试

AI 面试模块负责出题、追问和反馈。重点是流式输出，也就是 AI 不是等完整答案生成完才显示，而是一段一段返回，用户感觉更接近真人对话。

~~~text
前端发送问题 -> 服务端代理请求模型 -> SSE/流式返回 -> 页面逐字展示
~~~

这里最重要的工程点是安全。前端不直接拿模型 Key，而是统一请求后端代理。这样即使用户打开浏览器开发者工具，也拿不到真实密钥。

### 代码编辑器

面试里经常有算法题和代码改错题，所以项目接入 Monaco Editor。它的作用是让用户在页面里写代码，而不是只用文本框回答。

代码编辑器真正麻烦的地方不只是显示代码，而是状态同步：当前题目是什么，用户写到哪一步，AI 怎么读取代码并给建议，报告里怎么记录这次提交。

### 简历解析

简历解析的目标是让问题更贴近个人经历。用户上传 PDF 后，浏览器端用 pdfjs-dist 读取文本，再交给系统提取技术栈、项目经历和可能追问点。

这样 AI 不只问“什么是 TCP 三次握手”，还可以问“你在项目里为什么用 Redis Pub/Sub，消息丢了怎么办”。这种追问更接近真实面试。

### 能力雷达和复盘报告

面试结束后，系统会从技术深度、逻辑表达、系统设计、沟通清晰度等维度生成评价。

报告不是为了给一个漂亮分数，而是告诉用户下一次该补哪里。例如：回答能说出定义，但缺少工程场景；能写代码，但复杂度分析不清楚；能讲项目，但没有说明失败和取舍。

## 状态和数据怎么设计

面试过程里有很多状态：当前问题、历史回答、用户配置、错题本、报告、简历内容。如果全部靠组件一层层传，会很乱。

所以项目使用 Zustand 做集中状态管理，把跨页面共享的数据放在统一 store 里。长期记录则放到 IndexedDB，避免 localStorage 容量小、结构弱的问题。

~~~ts
const record = {
  question,
  answer,
  score,
  suggestion,
  createdAt,
};
~~~

这段数据结构看起来简单，但它决定了后续能不能做错题本、趋势图和历史复盘。

## 性能优化点

这个项目比较容易变重，因为 Monaco、Framer、PDF 解析和图表都不轻。优化思路主要有三类：

- 页面级懒加载：不是当前页面需要的功能，不进入首屏包。
- 依赖拆包：React、Monaco、Framer 这类大依赖单独拆 chunk。
- 本地持久化分页：历史记录不要一次性全读进内存。

性能优化不是“少做功能”，而是让重功能在正确的时候加载。

## 难点和踩坑

第一个难点是 AI 流式输出。流式响应要处理网络中断、内容追加、重复渲染和用户取消。如果每次收到一小段都粗暴更新整个页面，动画和输入都会卡。

第二个难点是安全代理。很多 AI Demo 会把 Key 放前端，这是不能上线的。这个项目把 AI 请求放到服务端代理里，同时通过环境变量管理 Key。

第三个难点是面试流程设计。AI 能回答问题，不代表能做好面试官。面试官需要追问、纠错、控制节奏、最后总结，这些都要通过提示词和状态控制配合完成。

## 如果继续优化

后续可以继续做几件事：

- 增加题库难度分层，让练习更稳定。
- 给报告增加历史趋势，看到自己有没有进步。
- 对系统设计白板增加更多组件约束和自动评分。
- 把简历解析结果变成可编辑结构，减少模型误判。
- 给代码题增加更多语言和测试用例。

## 项目总结

计算机面试大师最有价值的地方，不是用了 AI，而是把 AI 放进了一个完整训练流程里。它覆盖了提问、回答、追问、写代码、报告和错题复习，能展示前端工程、AI 接入、安全代理、数据持久化和产品闭环设计能力。
