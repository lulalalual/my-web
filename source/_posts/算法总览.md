算法不是背模板，而是训练一种拆问题的方式。看到一道题，先不要急着写代码，先问：数据规模多大，能不能暴力，重复计算在哪里，状态怎么变化，有没有单调性，能不能把问题拆成子问题。

## 复杂度是第一道筛选器

如果数据量只有 100，<code>O(n^2)</code> 可能完全够用；如果数据量是 100000，平方复杂度就很危险。复杂度不是为了面试好看，而是帮你判断方案能不能跑完。

一个常用判断是：一秒内大概能做千万级到上亿级简单操作。真实情况会受语言、机器和常数影响，但这个估算能帮你快速排除不靠谱方案。

## 双指针用于“两个位置一起移动”

双指针常见于有序数组、链表、字符串。它的核心是用两个位置共同维护一个关系，而不是反复嵌套循环。

~~~cpp
int left = 0;
int right = nums.size() - 1;
while (left < right) {
    int sum = nums[left] + nums[right];
    if (sum == target) break;
    if (sum < target) left++;
    else right--;
}
~~~

在有序数组两数之和里，小了就左指针右移，大了就右指针左移。每一步都能排除一批不可能答案，这就是双指针省时间的原因。

## 滑动窗口处理连续区间

滑动窗口适合处理“连续子数组或子串”的问题。窗口右边扩张收集数据，左边收缩排除不符合条件的数据。

~~~cpp
int left = 0;
for (int right = 0; right < n; ++right) {
    // 把 nums[right] 加进窗口
    while (窗口不合法) {
        // 移出 nums[left]
        left++;
    }
    // 此时窗口合法，更新答案
}
~~~

滑动窗口最重要的是定义清楚窗口里维护什么。可能是字符频次、当前和、最大最小值，也可能是不同元素个数。

## 二分查找依赖单调性

二分不是只能查数组，它查的是“答案空间里的单调性”。只要一个条件满足后，后面都满足，或者不满足后，后面都不满足，就可以考虑二分。

~~~cpp
int l = 0, r = n - 1;
while (l <= r) {
    int mid = l + (r - l) / 2;
    if (nums[mid] == target) return mid;
    if (nums[mid] < target) l = mid + 1;
    else r = mid - 1;
}
~~~

二分最容易错在边界。写代码前先明确区间是闭区间还是左闭右开，循环条件是什么，缩小区间时会不会漏掉答案。

## 递归是把问题交给更小的自己

递归不是玄学。它做的是把大问题拆成结构相同的小问题。写递归时只需要想清楚两件事：最小问题怎么处理，当前问题怎么依赖子问题。

~~~cpp
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}
~~~

树和图的 DFS 很适合递归，因为每个子树、每条分支都像一个缩小版问题。递归写不明白时，可以先画调用树，而不是盯着代码硬想。

## 回溯是在搜索中撤销选择

回溯适合排列、组合、子集、棋盘这类需要枚举选择的问题。它的套路是：做选择，进入下一层，撤销选择。

~~~cpp
void dfs(int start) {
    result.push_back(path);
    for (int i = start; i < nums.size(); ++i) {
        path.push_back(nums[i]);
        dfs(i + 1);
        path.pop_back();
    }
}
~~~

回溯不是无脑暴力。剪枝很重要。只要能提前判断某条路不可能得到答案，就应该尽早返回。

## 动态规划解决重复子问题

动态规划适合“有重叠子问题、最优子结构”的场景。它的难点不是写数组，而是定义状态。

以爬楼梯为例，到第 <code>i</code> 阶的方法数，只和第 <code>i-1</code> 阶和第 <code>i-2</code> 阶有关。

~~~cpp
std::vector<int> dp(n + 1);
dp[0] = 1;
dp[1] = 1;
for (int i = 2; i <= n; ++i) {
    dp[i] = dp[i - 1] + dp[i - 2];
}
~~~

写动态规划时按四步走：状态表示什么，选择有哪些，状态转移怎么写，初始值是什么。不要先背模板，先把“这个数组的每一格代表什么”说清楚。

## 贪心每一步都选当前最优

贪心适合局部最优能推出全局最优的问题。它写起来通常简单，但证明最难。比如区间调度里，优先选择结束最早的区间，就能留下最多空间给后面。

使用贪心前要确认这个选择不会影响未来更优解。如果只是“看起来合理”，不一定正确。能交换证明或反证，才算稳。

## 图算法先确定图的性质

图题先问：有向还是无向，有权还是无权，有没有负权，要求最短路、连通性还是拓扑顺序。

无权最短路通常用 BFS。有非负权最短路可以用 Dijkstra。有负权边要考虑 Bellman-Ford。依赖顺序可以用拓扑排序。连通分量可以用 DFS 或并查集。

图算法的难点不是名字，而是建图和状态。边怎么存，访问过的点怎么标记，路径代价怎么更新，这些决定代码是否正确。

## 算法练习的正确姿势

不要按题号刷到麻木。每做完一道题，都写下它属于哪个模型，为什么这个模型适合，关键边界是什么。如果同一个模型连续错三次，说明不是题没刷够，而是模型没理解透。

算法真正的目标不是记住一千道题，而是看到新题时能把它归到熟悉的思路里，再稳定地写出正确代码。
