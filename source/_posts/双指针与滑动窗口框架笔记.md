双指针和滑动窗口本质上都在做一件事：用两个边界维护一个“连续区间”，在移动边界时不断更新区间状态。

## 什么时候用

- 连续子数组、连续子串、区间最值、区间计数。
- 题目里出现“最短”“最长”“恰好包含”“至少包含”“至多包含”。
- 已排序数组中的配对、去重、原地压缩。

## 四种常见形态

- 左右同向：滑动窗口、快慢指针。
- 左右相向：有序数组两数之和、回文判断。
- 固定窗口：长度固定，右进左出。
- 可变窗口：条件满足时收缩，不满足时扩张。

## 核心模板

```cpp
int left = 0, right = 0;
while (right < n) {
    // 1. 扩张窗口
    add(nums[right]);
    right++;

    // 2. 条件满足时收缩
    while (window_is_valid()) {
        update_answer(left, right);
        remove(nums[left]);
        left++;
    }
}
```

这段模板的关键不是代码，而是两个问题：

- `add/remove` 维护了什么状态。
- `window_is_valid` 判断的是“已经满足”还是“已经不合法”。

## 容易写错的地方

- `right++` 放错位置，导致窗口实际范围和计算范围不一致。
- 在“至少满足”问题里，答案更新时机应该放在收缩前，而不是收缩后。
- `unordered_map` 统计字符频次时，减少到 0 以后是否要删键，需要和条件判断一致。
- 固定窗口题不能写成“满足条件就收缩”，否则窗口长度会乱掉。

## 复杂度理解

只要左右指针都只单调移动，每个元素最多进窗口一次、出窗口一次，总复杂度就是 `O(n)`。

## 典型变体

- 最长无重复子串：窗口内维护字符出现次数。
- 最小覆盖子串：窗口内维护目标字符欠账。
- 盛最多水的容器：左右相向，用短板移动证明最优性。
- 删除有序数组重复项：快慢指针把保留元素写回前面。
