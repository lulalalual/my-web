二分搜索要先想明白的不是代码，而是“单调性在哪”。如果题目没有单调性，二分就只是形式正确、逻辑错误。

## 两种常见场景

- 在有序数组中找目标位置。
- 在答案空间里找最小可行值或最大可行值。

## 半开区间模板

```cpp
int left = 0, right = nums.size(); // [left, right)
while (left < right) {
    int mid = left + (right - left) / 2;
    if (nums[mid] >= target) {
        right = mid;
    } else {
        left = mid + 1;
    }
}
```

半开区间好处是边界统一，适合写 `lower_bound` 和 `upper_bound` 风格问题。

## 答案判定模板

```cpp
long long left = min_answer, right = max_answer;
while (left < right) {
    long long mid = left + (right - left) / 2;
    if (check(mid)) {
        right = mid;
    } else {
        left = mid + 1;
    }
}
return left;
```

这里的 `check(mid)` 要表达“答案取到 mid 时是否已经可行”。只要 `check` 随着 `mid` 单调变化，二分就成立。

## 常见错误

- `mid = (left + right) / 2` 在大数场景下可能溢出。
- “找最大可行值”时还沿用“找最小可行值”的更新规则。
- `check(mid)` 写成了多条件混合，自己都说不清楚单调性是否真的存在。
- 最后返回 `left` 还是 `right` 没和区间定义对应起来。

## 实战判断标准

只要题目里出现：

- 最小的 `x` 使得某条件成立。
- 最大的 `x` 使得某限制不被突破。
- 有序数组里第一个大于等于、最后一个小于等于。

优先考虑二分。
