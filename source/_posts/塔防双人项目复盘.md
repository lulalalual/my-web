`塔防双人` 是一个基于 C++ 与 SDL2 的桌面塔防游戏。项目目录中已经形成较完整的游戏工程拆分：地图、关卡、敌人、炮塔、子弹、玩家、金币、UI 面板、资源管理和主循环都被独立管理。

## 技术栈

项目使用：

- SDL2：窗口、输入、渲染和事件循环。
- SDL_image：加载地图、角色、敌人、炮塔和特效图片。
- SDL_mixer：背景音乐、攻击、命中、胜负等音效。
- SDL_ttf：字体渲染。
- cJSON：读取 `config.json`、`level.json` 等配置。

## 游戏主循环

`GameManager` 负责整体生命周期。主循环维持 60 FPS，并使用硬件加速渲染器：

```cpp
while (!is_quit) {
    while (SDL_PollEvent(&event)) on_input();
    on_update(delta);
    SDL_RenderClear(renderer);
    on_render();
    SDL_RenderPresent(renderer);
}
```

主循环拆成输入、更新、渲染三段，便于把敌人、塔、防御子弹、玩家和 UI 面板都挂到统一的帧逻辑里。

## 玩法系统

### 地图与关卡

地图来自 `map.csv`，关卡来自 `level.json`。关卡配置定义了波次奖励、刷怪间隔、出生点和敌人类型。游戏启动时会预生成瓦片地图纹理，减少每帧重复拼接地图的成本。

### 炮塔系统

项目实现了三类塔：

- 弓箭塔：攻击间隔短，成长稳定。
- 斧手塔：攻击慢但单次伤害较高。
- 枪手塔：造价高，后期伤害更强。

每类塔都有多级 `interval`、`damage`、`view_range`、`cost` 和 `upgrade_cost` 配置，说明项目已经把数值和逻辑解耦。

### 敌人系统

敌人包含史莱姆、史莱姆王、骷髅、哥布林和哥布林祭司。不同敌人配置了生命、速度、伤害、奖励比例和回复能力，其中哥布林祭司具备范围恢复，给关卡节奏提供了变化。

### 玩家系统

玩家通过 `WASD` 移动，`J` 释放普通方向攻击，`K` 释放消耗 MP 的冲击技能。玩家还会拾取金币，并和敌人命中盒产生交互。

## 工程亮点

- `Manager<T>` 单例模板统一管理全局系统。
- `ResourcesManager` 集中加载纹理、音乐和音效，避免业务模块重复读取资源。
- `ConfigManager` 把地图、关卡和数值配置集中管理，便于调参。
- 地图纹理使用 `SDL_TEXTUREACCESS_TARGET` 预渲染，减少运行时瓦片绘制成本。
- 胜利/失败通过 `Banner` 和音效反馈，形成完整闭环。

## 复盘结论

这个项目最适合展示游戏工程拆分能力。它已经不是一个单文件 demo，而是包含数值配置、资源管理、帧循环、实体管理和 UI 面板的完整小型游戏。后续可以继续补充路径寻路、双人网络同步、关卡编辑器和性能分析。
